Versión 1.0 - Argentina

1. Definición de la Solución

Este proyecto implementa un sistema de Social Listening automatizado. La solución permite al cliente monitorizar temas, usuarios o URLs en la red social X, procesar la carga emocional (sentimientos) y visualizar resultados en un dashboard interactivo en tiempo real.

Valor de Negocio: Automatización total del análisis de reputación digital sin intervención manual en el código, facilitando la toma de decisiones estratégicas.

3. Arquitectura de Software y Seguridad

Se utiliza un patrón de Pipeline Desacoplado. La lógica de extracción es independiente de la visualización, permitiendo mantenibilidad y escalabilidad.

Diagrama de Flujo del Proceso

graph TD A[config.yaml] --> B[extraccion.py] B --> C{API X v2} C --> B B --> D[NLP Engine] D --> E[(sentimientos.db)] E --> F[visualizacion.py] F --> G[Dashboard HTML]

Protocolo de Seguridad

  • Secretos: Uso de .env para ocultar tokens de API.
  • Deduplicación: Restricción UNIQUE(tweet_id) en SQL para evitar inflar métricas.
  • Git: Uso estricto de .gitignore para proteger credenciales.

4. Marco Legal y Ético (Argentina)

El proyecto opera bajo el estricto cumplimiento de la Ley Nacional 25.326 de Protección de Datos Personales.

  • Anonimización: Los reportes finales no muestran datos que permitan la re-identificación de ciudadanos.
  • API Oficial: Se garantiza que no se realiza scraping, utilizando los canales autorizados por X Corp.
  • Fines Estadísticos: Los datos se procesan con el fin único de análisis de tendencias públicas.
